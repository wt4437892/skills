# Effective Review Principles

Use this note when adjusting `java-senior-reviewer` strategy.

## Core rules

1. Retrieval before review
- Let the user answer from memory before showing the answer.
- Do not show the answer, options, or long hints too early.

2. Immediate feedback
- After the retrieval attempt, show corrective feedback immediately.
- Explanatory feedback is better than only marking right or wrong.

3. Low-stakes review
- Treat review as a learning activity, not an exam.
- Keep scoring for calibration and scheduling, not pressure.

4. Minimal scaffolding
- If the user gets stuck, prefer a short retrieval cue first.
- Avoid turning every review item into multiple-choice.

5. Corrective retrieval
- If the user answers “不知道” or performs poorly, show the answer and then ask for a very short restatement from memory.

6. Interleaving
- When multiple domains are due on the same day, alternate domains when possible instead of batching one domain for too long.
