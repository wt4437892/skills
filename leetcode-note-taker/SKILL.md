# LeetCode 算法题笔记助手

帮助用户快速创建结构化的 LeetCode 算法题笔记。

## 使用场景

当用户说：
- "帮我记录 LeetCode 239 滑动窗口最大值"
- "写一下两数之和的题解"
- "记录一下这道算法题的笔记"

## 笔记结构

生成的笔记包含以下部分：

### 必需部分

1. **题目标题**：`## LeetCode [编号]. [题目名称]`
2. **题目描述**：清晰描述题目要求
3. **示例**：用代码块展示输入输出示例
4. **解题思路**：核心算法思路，用简洁的语言说明
5. **Java 代码**：完整可运行的代码实现
6. **案例代码解读**：用具体示例完整追踪代码执行过程
   - 列出初始化状态（所有关键变量的初始值）
   - 逐步展示每次循环的变量变化
   - 标注窗口/指针的移动过程
   - 说明关键判断条件的触发时机
   - 展示最终结果的计算过程
7. **复杂度分析**：时间和空间复杂度

### 可选部分（根据题目复杂度添加）

7. **面试易错点**：常见错误和注意事项
8. **面试追问/回答要点**：面试中可能的追问及标准回答
9. **延伸知识点**：相关算法和数据结构
10. **实战应用场景**：实际工程中的应用

## 工作流程

1. **收集信息**：询问用户题目编号、名称、分类（如双指针、哈希表、滑动窗口等）
2. **确认路径**：根据分类确定保存路径，格式为 `LeetCode100/[分类]/LeetCode-[编号]-[题目名称].md`
3. **生成笔记**：按照结构模板生成笔记内容
4. **保存文件**：将笔记保存到指定路径

## 代码规范

遵循用户的 Java 代码规范：
- 使用 import 语句，不使用全限定类名
- 代码简洁，只包含核心逻辑
- 添加必要的注释说明关键步骤

## 示例对话

**用户**：帮我记录 LeetCode 239 滑动窗口最大值

**助手**：
1. 题目编号：239
2. 题目名称：滑动窗口最大值
3. 题目分类：滑动窗口
4. 保存路径：`LeetCode100/滑动窗口/LeetCode-239-滑动窗口最大值.md`

请提供题目描述和你的解题思路，我会帮你生成完整的笔记。

## 注意事项

- 如果用户没有提供分类，根据题目特点建议合适的分类
- 如果用户只提供题目编号，帮助查找题目名称
- 代码必须是完整可运行的，包含必要的 import 语句
- 复杂度分析要准确，说明为什么是这个复杂度
- 面试易错点要具体，不要泛泛而谈
