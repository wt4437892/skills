# Java 面试高频冲刺参考

只在用户明确要求 `高频`、`冲刺`、`突击`、`短期突破`，或表达“最近要面试、想快速提分”时读取对应领域。
目标不是覆盖所有知识点，而是优先命中“高频、可搜、短期收益高”的题。

- 索引为什么能加速查询，回表和覆盖索引怎么理解

- 事务四大特性在 InnoDB 里怎么落地
- MVCC 是怎么实现可重复读的
- 行锁、间隙锁、临键锁分别解决什么问题
- redo log、undo log、binlog 的作用和区别
- MySQL 主从复制原理，以及为什么会有主从延迟

## Redis

- Redis 为什么快
- Redis 常见数据结构的底层实现和适用场景

- Redis 持久化 RDB 和 AOF 怎么选
- Redis 主从、哨兵、集群各自解决什么问题
- BigKey、热 Key 会带来什么问题，怎么发现和治理

## Spring / Spring Boot
